Quick Start in 3 Steps

1️⃣ Install CURD

go install github.com/dmcbane/curd/v2@v2.2.0

Or download the binary

2️⃣ Add to Shell

Add to ~/.bashrc (Bash):

function curr() {
  D=$(curd "$@")
  cd "${D}"
}

# Tab completion of saved keywords
if [ -n "$BASH_VERSION" ]; then
  _curr_complete() {
    COMPREPLY=()
    if [ "$COMP_CWORD" -eq 1 ]; then
      COMPREPLY=($(compgen -W "$(curd ls -k)" \
        -- "${COMP_WORDS[COMP_CWORD]}"))
    fi
  }
  complete -F _curr_complete curr
fi

Using another shell? See the installation guide for Zsh, Fish, PowerShell, and csh/tcsh.

3️⃣ Start Using

# Save current directory
curd save project

# Jump to it anytime
curr project
## Why CURD? Unlike tools that try to predict where you want to go based on usage patterns, CURD gives you complete control. You explicitly save the directories you care about and access them with memorable keywords you choose. ### Perfect for: - 👨‍💻 **Developers** - Jump between projects, configs, and documentation - 🎨 **Designers** - Quick access to asset folders and project files - 📊 **Data Scientists** - Navigate between datasets and notebooks - 🔧 **System Administrators** - Fast access to log directories and configs - 📝 **Writers** - Organize and access different writing projects ## Example Workflow ```bash # Morning setup cd ~/work/backend && curd save be cd ~/work/frontend && curd save fe cd ~/work/database && curd save db cd ~/.config && curd save config # Throughout the day curr be # Jump to backend curr fe # Jump to frontend curr db # Jump to database curr config # Jump to config files # End of day cleanup curd list # See all saved paths curd clean # Remove non-existent paths ``` ## What's New in v2.2.0

🐚 Keyword Tab Completion for curr

The curr integration scripts now include tab completion of your saved keywords in every supported shell — Bash, Zsh, Fish, PowerShell, and csh/tcsh — with new ready-to-use curr.zsh and curr.csh scripts.

## What's New in v2.1.0

🐚 Shell Completion Generator

Generate completion scripts for bash, fish, and zsh with curd completions [<shell>]. Omit the shell to auto-detect it from $SHELL. The old static curd_completion.bash file is gone.
